Healthy Eats Redesign

Redesigned a production food-ordering app to simplify decision-making and remove checkout friction, using system-driven design to improve clarity, consistency, and flow.

Contact
Project:  Healthy Eats
Date:  2025
Role:  Product UI Redesign

Problem statement

A healthy food ordering app suffered from weak visual hierarchy and inconsistent UI patterns, forcing users to think too much for simple tasks. Usability testing with 5 participants revealed that unclear CTAs, overlooked pricing, and dense layouts increased cognitive load and slowed checkout, leading to hesitation and abandonment.

Challenge

The challenge was to reduce cognitive load, improve scanability of products and prices, create a scalable design system, and increase checkout confidence through clear hierarchy and predictable flows, without adding visual clutter.

Solution

We delivered a system-driven redesign: fixed content → price → action hierarchy, one primary CTA per screen, card-based layouts for quick scanning, and a linear checkout flow. Reusable components and consistent patterns reduced decision-making time and increased confidence.

How We Solved It

Clear Hierarchy & Scanability

Fixed content → price → action pattern on every card. Bottom navigation and card-based layouts improve scanability so users compare options and identify next steps without reading every detail.

Linear Checkout Flow

Wireframes established a predictable progression: basket → payment → delivery. Fewer steps and clear next-step CTAs reduce hesitation and increase checkout confidence.

Problem Analysis

Testing revealed weak hierarchy, unclear CTA priority, and pricing overlooked. Dense layouts and inconsistent navigation increased cognitive load and slowed task completion.

Design System

Reusable components and consistent typography, colors, and patterns scale to new features without redesign. When patterns repeat, users apply prior knowledge instead of re-learning.

Mobile Experience

Same hierarchy and patterns on mobile. One primary action per screen and predictable flows reduce cognitive load and build confidence on small screens.

One Primary Action

Each screen has one clear primary CTA; secondary actions stay accessible but subordinate. This removes hesitation about which button to tap and reduces decision paralysis.

Figma Prototype

Interactive prototype showcasing the complete user experience and design flow.

Transformation

Before vs After

The redesign addressed core structural issues. Weak hierarchy became clear hierarchy. Inconsistent patterns became predictable flows. Fragmented experiences became a scalable system.

Before After

Before: Cluttered layouts with weak hierarchy made scanning difficult. After: Clear structure with fixed content → price → action pattern enables quick comparison.

Before After

Before: Fragmented checkout with unclear progression caused hesitation. After: Linear, predictable flow with clear next steps reduces cognitive load.

Outcome

The system-driven redesign reduced cognitive load and improved checkout clarity. Fixed content → price → action hierarchy and one primary CTA per screen eliminated hesitation. Usability testing with 5 participants informed every decision; consistent patterns and predictable flows enabled faster task completion and higher confidence.

Reduced cognitive load: Clear hierarchy and repeatable patterns meant users applied prior knowledge instead of re-learning, speeding up browsing and comparison.

Improved checkout confidence: Linear flow with clear next steps and persistent order summary reduced hesitation and abandonment.

A scalable design system with reusable components extends to new features without redesign, reducing design debt and keeping the experience consistent.
